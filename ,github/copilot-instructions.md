# Home Assistant JavaScript and Node.js Guidelines

# Purpose
This repository uses Lit for building web components. Always prioritize native browser features over external UI frameworks.


## 1. Core Principles
- Language: Modern JavaScript (ESM)
- Target Environment: Home Assistant Core (Node.js environments).
- Styling: Use CSS custom properties matching the HA theme (e.g., `--primary-color`, `--accent-color`).
- Use standard modern JavaScript (ES6+), leveraging `import` and `export` over CommonJS.
- Semicolons are required at the end of statements.
- Integration: Leverage the global `applyTo` window objects where applicable for HA state management.
- Stick to standard JavaScript and modern browser features (Custom Elements, Shadow DOM, CSS custom properties).
- Avoid non-standard TypeScript features (like namespaces) unless explicitly required by the project.
- The Version should be bumped up for each change.

## 2. Home Assistant Communication
- All backend communication to Home Assistant MUST use the `homeassistant-js-websocket` API.
- Do not hardcode entity IDs. Parameterize them so the user can easily swap them out.
- Always implement proper error handling (e.g., `try...catch` blocks) for all WebSocket and REST API calls.
- When querying state, fetch historical data or logbook entries asynchronously, and cache responses appropriately for frequently accessed data.
- Exception will be when it is required to connect with frigate while bypassing HA

## 3. Lit.js UI Conventions
- Base class: Always extend `LitElement`.
- Imports: Use standard ESM imports. For relative files, ensure the `.js` extension is included.
- Decorators: Avoid complex decorators; use standard `static get properties()` and standard JavaScript for simplicity.
- Structure: Separate your logic clearly into `static get styles()` and `render()`.
- Animations: Utilize standard transition wrappers and `--ha-animation-duration-*` CSS variables.
- Use class decorators like `@customElement('my-element')` and `@property()` for reactive properties.
- Ensure the `render()` method returns a `TemplateResult` generated by the `html` tag.
- Implement `static styles` using the `css` tag for encapsulation.
- Always use the `constructor()` for setting internal state (not for properties wired to `@property()`).

## 4. Node.js Backend & API
- Environment: Target Node.js $20.19.0$ or newer.
- Modules: Use native `import/export` instead of CommonJS.
- Dependencies: Whenever possible, utilize `node-fetch` or the native `fetch` API when communicating with the Home Assistant REST API.
- Error Handling: Standardized `try/catch` blocks should be implemented around all asynchronous network requests to Home Assistant.

## 5. Node.js & Automation Rules
- For standalone Node.js automations, utilize the `homeassistant-jsengine` package conventions.
- When writing scripts, always handle real-time state changes by subscribing to Home Assistant's event stream (e.g., `state_changed`).
- Prefer the use of JS `async/await` patterns over nested callbacks to keep automation logic readable.

## 6. Formatting & Code Quality
- Add JSDoc comments for all major functions, explaining their purpose, parameters, and return values.
- Adhere strictly to the requested asynchronous paradigm—do not block the event loop with synchronous state polling.

# 7. Component Design
- Keep components small, focused, and compliant with the Single Responsibility Principle.
- Name custom elements using kebab-case (e.g., `<user-profile>`).
- Use descriptive property names and camelCase for property definitions.
- Write JSDoc comments for public APIs and complex custom methods.

# 8. Security
- Never expose sensitive private keys or seed phrases in the code.
- Always sanitize user input before rendering it in the DOM (to prevent XSS attacks).
- Handle asynchronous operations (such as API calls or promises) safely with proper error boundaries and loading states in templates.


## 9. Output Examples
Example of a basic Lit.js component structure for Home Assistant:
```javascript
import { LitElement, html, css } from 'lit';

class MyHaComponent extends LitElement {
  static get properties() {
    return {
      hass: { type: Object },
      config: { type: Object }
    };
  }

  static get styles() {
    return css`
      :host {
        display: block;
        padding: 16px;
        background-color: var(--card-background-color);
        color: var(--primary-text-color);
      }
    `;
  }

  render() {
    return html`
      <div>Hello Home Assistant!</div>
    `;
  }
}
customElements.define('my-ha-component', MyHaComponent);
```
